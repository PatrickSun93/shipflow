# sf-regen-index

Rebuild the index from scratch.

## Arguments

None. Always operates on `docs/shipflow/index.md` in the current repo.

## Steps

1. **Check for init.** If `docs/shipflow/` doesn't exist, tell the user to
   run `/sf-init` first and stop.

2. **Scan four directories, frontmatter only** (don't read bodies):
   - `docs/shipflow/briefs/*.md` — capture `id`, `slug`, `status`, H1 title
   - `docs/shipflow/stories/*.md` — capture `id`, `brief`, `status`, `size`,
     `type` (for TINY records), H1 title
   - `docs/shipflow/decisions/*.md` — capture `id`, H1 title
   - `docs/shipflow/releases/*.md` — capture version (filename), H1 title,
     date from frontmatter if present

   **Never** read `docs/shipflow/archive/`. Archived items stay cold.

3. **Sort each list:**
   - Briefs: id ascending
   - Stories: id ascending (TINY and STORY interleave by numeric id)
   - ADRs: id ascending
   - Releases: version descending (newest first)

4. **Overwrite `docs/shipflow/index.md`** with this layout:

   ```markdown
   <!-- Auto-regenerated by /sf-regen-index. Don't edit by hand — changes are lost on the next regen. -->

   # ShipFlow index

   _Last regenerated: <yyyy-mm-dd>_

   ## Briefs

   - [<id>](briefs/<filename>) — <title> — `<status>`
   - ...

   ## Stories

   - [<id>](stories/<filename>) — <title> — `<status>` — brief: <brief-id>
   - [TINY-<id>](stories/<filename>) — <title> — `<status>` — tiny
   - ...

   ## ADRs

   - [<id>](decisions/<filename>) — <title>
   - ...

   ## Releases

   - [<version>](releases/<filename>) — <date>
   - ...
   ```

   For any section with zero items, keep the H2 header and write
   `_None yet._` underneath.

5. **Report to the user:**
   - Path written
   - Counts per section (e.g. "4 briefs, 12 stories, 3 ADRs, 2 releases")

## Hard rules

- **Pure overwrite.** Any manual content in `index.md` is lost. That's by
  design — `index.md` is a derived artifact. Put notes that should survive
  in `stack.md` or the relevant brief/story/ADR.
- **Never read archive.** Cold layer stays cold.
- **Frontmatter only.** Don't open file bodies — wasteful, and the index
  shape doesn't need them.
- **No agent spawn.** This is deterministic file work. The skill runs the
  scan and the write directly.
